# SELECT 기본 조회

## 핵심 정리

- SELECT는 테이블에서 데이터를 조회할 때 사용
- `*`는 모든 컬럼 조회
- 필요한 컬럼명만 적으면 일부 컬럼만 조회
- WHERE는 조건에 맞는 행만 조회
- ORDER BY는 조회 결과 정렬

## 실습 내용

- 전체 컬럼 조회
- 필요한 컬럼만 조회
- 급여 조건 조회
- 급여 기준 내림차순 정렬

## 복습 메모

- pgAdmin4 실행 -> PostgreSQL / Databases 우클릭 눌러 Create -> Database... 로 이름 지정 후 데이터베이스 생성
- 이름: kh_sql_practice
- 만들어진 kh_sql_practice 우클릭 -> Query Tool 열고 실습 쿼리 실행

## 학습 중 생긴 의문

### 의문: `*`는 단순히 편의를 위해 만들어진 것인가? 컬럼을 모두 적는 것과 성능 차이가 있는가?

#### 답변: 편의를 위해 만들어진 것이 맞으며, 필요한 컬럼이 전체 컬럼이라면 성능 차이는 거의 없으나 변경에 취약하다.

- 나중에 컬럼이 추가되면 결과가 자동으로 바뀜
- 애플리케이션에서 예상하지 않은 데이터까지 받게 됨
```
DTO에 id, name, address가 존재
*를 통해 DB에서 조회된 민감정보 password 등
DTO에 존재하지 않아 화면에 보이지 않아도 개발자의 로그 등을 통해 유출 가능
```
- 컬럼 순서에 의존하는 코드가 있으면 문제 발생 가능
- 어떤 컬럼을 사용하는지 코드만 보고 알기 어려움
