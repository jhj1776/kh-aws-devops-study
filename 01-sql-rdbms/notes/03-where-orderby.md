# WHERE 조건 조회와 ORDER BY 정렬

## 핵심 정리

- `WHERE`는 조건에 맞는 행만 조회할 때 사용
- 비교 연산자로 숫자, 문자열, 날짜 조건을 지정
- `AND`는 모든 조건을 만족하는 행 조회
- `OR`는 여러 조건 중 하나라도 만족하는 행 조회
- `IN`은 여러 값 중 하나에 해당하는 행 조회
- `LIKE`는 문자열 패턴 조회
- `ORDER BY`는 조회 결과 정렬
- `ASC`는 오름차순 정렬
- `DESC`는 내림차순 정렬

## 실습 내용

- 급여 조건 조회
- 부서 ID 조건 조회
- AND 조건 조회
- IN 조건 조회
- LIKE 문자열 조회
- 날짜 조건 조회
- 급여 기준 내림차순 정렬
- 입사일 기준 오름차순 정렬

## 복습 메모

- 문자열과 날짜 값은 작은따옴표로 감싸서 작성
- 숫자 값은 작은따옴표 없이 비교
- 조건이 여러 개일 때는 AND, OR를 사용
- 여러 값 중 하나를 비교할 때는 IN을 사용
- 문자열 일부를 검색할 때는 LIKE와 `%` 사용
- ORDER BY는 보통 SELECT문의 마지막에 작성
### SQL 작성 순서와 처리 순서

SQL은 보통 아래 순서로 작성한다.

```sql
SELECT 컬럼
FROM 테이블
WHERE 조건
ORDER BY 정렬기준;
```

- 실행은 순서는 FROM에서 테이블을 정하고, WHERE로 조건에 맞는 행을 거른 뒤, SELECT로 필요한 컬럼을 조회하고, 마지막에 ORDER BY로 정렬

## 학습 중 생긴 의문

### 의문: LIKE에서 `%`와 `_`는 무엇을 의미하는가?

#### 답변: `LIKE`는 문자열 패턴을 비교할 때 사용하며, `%`는 0개 이상의 문자를 의미한다.

- LIKE 'Kim%': Kim으로 시작하는 값
- LIKE '%Kim': Kim으로 끝나는 값
- LIKE '%Kim%': Kim이 포함된 값
- LIKE '_i%': 두 번째 글자가 i인 값
- %: 0개 이상의 문자
- _: 정확히 1개의 문자
