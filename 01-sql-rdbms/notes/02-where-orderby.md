\# WHERE 조건 조회와 ORDER BY 정렬



\## 핵심 정리



\- WHERE는 조건에 맞는 행만 조회할 때 사용

\- 비교 연산자로 숫자, 문자열, 날짜 조건을 지정

\- AND는 모든 조건을 만족하는 행 조회

\- OR는 여러 조건 중 하나라도 만족하는 행 조회

\- IN은 여러 값 중 하나에 해당하는 행 조회

\- LIKE는 문자열 패턴 조회

\- ORDER BY는 조회 결과 정렬

\- ASC는 오름차순 정렬

\- DESC는 내림차순 정렬



\## 실습 내용



\- 급여 조건 조회

\- 부서 ID 조건 조회

\- AND 조건 조회

\- IN 조건 조회

\- LIKE 문자열 조회

\- 날짜 조건 조회

\- 급여 기준 내림차순 정렬

\- 입사일 기준 오름차순 정렬



\## 복습 메모



\- 문자열과 날짜 값은 작은따옴표로 감싸서 작성

\- 숫자 값은 작은따옴표 없이 비교

\- 조건이 여러 개일 때는 AND, OR를 사용

\- 여러 값 중 하나를 비교할 때는 IN을 사용

\- 문자열 일부를 검색할 때는 LIKE와 `%` 사용

\- ORDER BY는 보통 SELECT문의 마지막에 작성
